# Examples - SaaS

This folder contains reference projects for the SaaS sector.

## Purpose

Projects stored here serve as design and implementation references for new SaaS clients.

Includes: B2B SaaS, developer tools, productivity software, platform products.

## When to Use

- Before starting a new SaaS project, review existing examples for pattern reference.
- Use examples to calibrate dials (Variance, Motion, Density) for the sector.
- Do not copy content from examples. Use them for structural and design reference only.

## Sector Notes

- SaaS projects are high variance (7-9). Layout experimentation is expected.
- Motion dial is typically high (6-9). Interactivity signals product quality.
- Primary conversion: free trial, demo booking, or waitlist signup.
- Social proof: logos of known companies using the product (never fake).
- Avoid generic SaaS language: "seamless", "powerful", "all-in-one".

## Adding a New Example

After completing a SaaS project, the EscalaSites team may promote it to this folder as a reference.

Only completed, QA-approved projects should be added as examples.

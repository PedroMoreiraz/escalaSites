# Color Principles

## Color Usage & Scarcity
- **Color is Scarce**: Use color only for semantic meaning, state changes, or as a single focal accent.
- **Max 1 Accent**: Stick to a single accent color for the entire page. Saturation < 80% by default.
- **Theme Lock**: The page must have ONE locked theme (Light, Dark, or Auto). Never mix dark mode sections inside a light mode page (or vice versa).

## The Lila Rule (Anti-AI-Purple Bias)
- **Ban on Default Slop**: The "AI Purple / Blue glow" aesthetic is strictly discouraged.
- **No Random Neons**: Do not use automatic purple button glows or neon AI gradients unless explicitly demanded by the brand. Use neutral bases (Zinc/Slate/Stone) with high-contrast, singular accents (Emerald, Electric Blue, Deep Rose).

## Premium Consumer Palette Ban
- **The Cliché**: The default AI palette for premium physical goods (warm beige/cream backgrounds + brass/ochre accents + espresso dark text) is BANNED.
- **Alternative Rotations**:
  - *Cold Luxury*: Silver-grey + Chrome + Smoke
  - *Forest*: Deep green + Bone + Amber
  - *Black and Tan*: True off-black + Warm tan
  - *Monochrome Pop*: Pure black & white + one bright electric accent (e.g., Hot Pink)

## Contrast Rules
- **A11y Button Contrast**: Every CTA must pass WCAG AA (4.5:1 ratio). White text on white background, or transparent ghost buttons over photographic backgrounds without a scrim, are banned.
- **Form Contrast**: Inputs, placeholders, focus rings, and labels must all pass contrast checks against their container background.
- **Monochrome & Pastels**: For minimalist UIs, rely on pure white or warm bone canvas, pure off-black typography, and heavily desaturated, washed-out spot pastels for tags (e.g., Pale Red `#FDEBEC` with `#9F2F2D` text).
